afirmation

This command must be used in conjunction with the toggle, on, or off commands. When used with toggle it toggles the afirmpref variable with a value of 1 meaning the afirmation is turned on and a value of 0 meaning the afirmation is turned off. This variable is saved in the jarvis.ini file. It dictates whether JARVIS will say "i am sorry, i didnt understand that, try again" when you say a string that contains no commands. When used with the on command, the afirmpref variable is set to 1 regardless of what its value was before. When used with the off command, the afirmpref variable is set to 0 regardless of what its value was before.

Future Additions

This section just contains ideas that I would like to incorporate into JARVIS in futre updates. I will be working on the following ideas:

A JARVIShome program that would act like a home that can control many aspects of JARVIS, you would be able to send JARVIS commands (which would solve that pesky "I have lost communication with JARVIS and he is in 'hide' mode" problem), run all the tools, easily manage your schedule, reminders, timers, and alarms, edit the settings variables more easily, and ultimately make JARVIS easier to use.

Give JARVIS the ability to learn what programs you have installed on your computer and run them by voice. This could be done by reading the registry or looking the the Start\All Programs folder.

Fix the 'timer' command so that you can ask for timers that are after midnight (this will be done by giving JARVIS its own timer to keep track of), and allow you to use a conjunction of two or more time measurements.

Add 'reminder' and 'schedule' commands.

Give JARVIS the ability to search for and play music files by voice.

Give JARVIS the ability to switch between windows using an "Alt-Tab" like function.

Give JARVIS the ability to use the 'transparency' command on all running programs instead of just itself.

Add a 'macro' command that lets JARVIS record and recreate keystrokes and mouse movent.

Add a messenger type protocal to JARVIS and the server that allows a user to say, for example, "Tell John I said hi." Then John's JARVIS would tell John, "Zack says hi". This would also encompass the creation of a type of address book for your friends JARVIS id's, and an id system run by the server.
